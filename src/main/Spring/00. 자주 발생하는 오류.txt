0. 콘솔에서 ERROR 메시지 이하를 확인하여, 구글링해서 이유 찾기

1. Spring 서버는 2개 이상 실행할 수 없다.
    -> port가 중복되므로, 오류 발생
    -> port 자동 종료가 안 되는 경우
        방법1) 컴퓨터 재시작
        방법2) 직접 port 종료하기
            cmd -> 'netstat -ano'
                -> 0.0.0.0:8080 의 오른쪽 PID 확인
                -> 'taskkill /f /pid [PID]' 예) 'taskkill /f /pid 6420'

2. 스프링에서 resources 폴더와 webapp 폴더는 모든 AppStart가 공유한다.

3. JSP는 무조건 webapp 폴더 내 자료만 인식한다.

4. 프론트엔드 ( JSP/CSS/JS )들을 일부 수정할 경우, 브라우저가 식별 안 되는 경우도 존재한다.
    -> Ctrl + F5로 강력 새로고침 권장

5. 모든 프로그래밍 언어는 '미리 만들어진 단어'로 변수명/함수명/클래스명으로 지정할 수 없다.
    -> JAVA는 키워드 오류가 바로 발생
    -> JS는 키워드 오류가 Console에서 발생 -> Console을 확인

6. 프론트엔드에서 경로 설정
    1) http://localhost:8080 (절대경로) vs / (상대경로)
        [오류X] <jsp:include page="http://localhost:8080/header.jsp"></jsp:include>
        [오류X] <jsp:include page="/header.jsp"></jsp:include>
        [오류O] <jsp:include page="header.jsp"></jsp:include>
    
    2) fetch : Controller에서 설정한 경로 확인
        [오류X] fetch( "http://localhost:8080/member/login", option )
        [오류X] fetch( "/member/login", option )
        [오류O] fetch( "member/login", option )
