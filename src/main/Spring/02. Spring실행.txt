[ 스프링 실행방법 ]
1. 폴더 내 AppStart 클래스 생성
2. 클래스 위에 @SpringBootApplication 자동완성
    -> 자동완성이 안될경우 : Gradle 설정이 안된상태
3. main 메소드 안에 SpringApplication.run( AppStart.class ); 작성
4. 실행 후 확인 : http://localhost:8080/
    -> 정상 화면 : Whitelabel Error Page
    -> 비정상 화면 : '사이트에 연결할 수 없음'

[ Spring ]
1. 자바 기반의 엔터프라이즈급 애플리케이션 개발을 위한 프레임워크
2. 전자정부프레임워크(대한민국 개발 표준)에 사용된다.
3. 특징 : IOC/DI(객체의존성), AOP(로직분리 모듈화), MVC2 3tier(필수), 확장성
[ SpringBoot ]
1. 스프링을 빠르게 사용할 수 있도록 자동설정기반의 스타터 프레임워크
2. 복잡한 설정을 제공받는다.
3. 내장 Tomcat을 제공한다.
    -> Tomcat이란? 자바에서 만든 웹개발 관련 라이브러리를 포함한 소프트웨어
    -> Starting Servlet engine: [Apache Tomcat/10.1.43]
[ Gradle ]
1. 자바 기반의 빌드 자동화 소프트웨어
2. 특징 : 의존성 관리, 배포, 패키징을 자동처리


- 코드 : 명령어
- 라이브러리 : 재사용 가능한 코드를 모아서 .jar 확장자로 만든 집합, 미리 만들어진 코드모음
- 프레임워크 : 라이브러리 + 개발에 필요한 환경설정
